# [Attriax](https://attriax.com) Flutter SDK

`sdk-flutter/` is the federated Flutter SDK workspace for Attriax.

This workspace is the primary SDK implementation and the behavioral reference
for the secondary `sdk-js/` and `sdk-unity/` runtimes.

It contains the publishable packages plus a public example app.

The implementation packages are published so the federated Flutter plugin can
resolve cleanly from pub.dev. Most apps should import `attriax_flutter`, not the
platform packages directly.

## Workspace Contents

- `attriax/` — main public package
- `attriax_api_client/` — generated internal transport client from the API SDK contract
- `attriax_flutter_platform_interface/` — shared models and platform contract
- `attriax_flutter_android/` — Android implementation
- `attriax_flutter_ios/` — iOS implementation
- `attriax/example/` — public example app

## Platform Support

- Android
- iOS
- Web
- Windows
- macOS
- Linux

Android and iOS provide native collection and built-in deep-link listeners. Web resolves the initial URL in Dart. Desktop platforms currently require manual URI forwarding from the host app.

## Getting Started

### Install dependencies

```bash
cd sdk-flutter
flutter pub get
```

### Run analysis and tests

```bash
cd sdk-flutter
dart analyze
cd attriax && flutter test
cd ..\attriax\example && flutter test
cd ..\..\attriax_flutter_platform_interface && flutter test
cd ..\attriax_flutter_android && flutter test
cd ..\attriax_flutter_ios && flutter test
```

The workspace root does not own a top-level `test/` directory, so `flutter test`
should be run from the package directories that contain tests.

### Regenerate the generated SDK client

From the workspace root:

```bash
npm install
npm run sdk:flutter:generate
```

For a faster regeneration pass without analyzer/test validation:

```bash
npm run sdk:flutter:generate:fast
```

To rerun only the supported validation checks:

```bash
npm run sdk:flutter:validate
```

See [SDK_CLIENT_GENERATION.md](SDK_CLIENT_GENERATION.md) for the full workflow,
what is autogenerated, and why the generated client currently lives in its own
internal package.

### Run the public example

```bash
cd sdk-flutter\attriax\example
flutter run

# For the full SDK flow, provide a real token.
flutter run --dart-define=ATTRIAX_APP_TOKEN=...
```

For Flutter web runs, add the served origin to the Attriax app setup page's
Web allowed browser origins before testing. Local runs commonly use
`http://localhost:3000`.

### Run the internal tester

```bash
cd flutter-internal-tester
flutter pub get
flutter run --dart-define=ATTRIAX_APP_TOKEN=... --dart-define=ATTRIAX_API_BASE_URL=...
```

## Development Notes

- Prefer `await attriax.init()` as the default startup path.
- Keep `sdk-js/` and `sdk-unity/` behaviorally aligned with this Flutter implementation unless a platform-specific divergence is explicitly documented.
- Use `flutter pub get` at the workspace root unless you intentionally need isolated package resolution.
- Keep package-level `analysis_options.yaml` files in sync with the workspace root so dry-run publish checks and standalone package CI see the same lint configuration.
- Keep the public example simple and package-focused.
- Use the internal tester for richer QA scenarios such as app-open tracking, manual deep-link resolution checks, install-referrer inspection, event submission, and identification flows.

	the configuration they ship.

## Cross-SDK Surface Notes

| Capability | Flutter | Web | React | Unity |
| --- | --- | --- | --- | --- |
| Page view tracking | `recordPageView(...)` | `recordPageView(...)` | `useAttriaxPageView(...)` or `attriax.recordPageView(...)` | `TrackPageViewAsync(...)` |
| Error tracking | `recordError(...)` | `recordError(...)` | `useAttriax().attriax.recordError(...)` | `RecordErrorAsync(...)` |
| Dynamic link creation | `createDynamicLink(...)` | `createDynamicLink(...)` | `useAttriax().attriax.createDynamicLink(...)` | `CreateDynamicLinkAsync(...)` |
| Manual deep-link reporting | `recordDeepLink(...)` | `recordDeepLink(...)` | `useAttriax().attriax.recordDeepLink(...)` | `RecordDeepLinkConversionAsync(...)` |
| ATT status/request | `getTrackingAuthorizationStatus()` and `requestTrackingAuthorization()` on Apple platforms | not supported | not supported | not supported |
